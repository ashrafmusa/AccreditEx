## 2) Getting Started & Sign In

### 2.1 Sign In (60 seconds)

**Steps:**
1. Open https://accreditex.web.app in any modern browser (Chrome, Edge, Safari, Firefox)
2. Enter your **email address** (provided by your admin)
3. Enter your **password**
4. Click **Login**
5. You will see the **Dashboard** within 2-3 seconds

**Troubleshooting Sign-In:**
| Issue | Solution |
|-------|----------|
| "Unknown email" | Confirm admin sent you an invite; check spam folder |
| "Wrong password" | Click **Forgot Password** and reset via email link |
| "Login loops" or page hangs | Clear browser cache (`Ctrl+Shift+Del`), then sign in again |
| Device says "untrusted" | Contact your IT/admin — use verified IP/VPN if required |

### 2.2 First Login Checklist (5 minutes)

After signing in:
1. ☐ Review your assigned role (hover over **Profile icon** → **Profile**)
2. ☐ Set your **Language** preference (Settings → switch English/Arabic)
3. ☐ Set your **Theme** (Settings → Light/Dark mode)
4. ☐ Pin **3 favorite modules** by clicking star icon on left sidebar
5. ☐ Bookmark https://accreditex.web.app for quick access

### 2.3 Language and Theme Settings

**Change Language:**
- Click **Settings** (bottom left or Profile menu)
- Select **Language** → **English** or **العربية (Arabic)**
- Interface automatically updates; RTL layout applies for Arabic
- Reload page if text doesn't refresh immediately

**Change Theme (Light/Dark):**
- Click **Settings** → **Visual Settings**
- Toggle **Dark Mode** on/off
- Adjust fonts and contrast if needed for accessibility

### 2.4 Role-Based Access Explained

Your **role** determines what modules you see:

| Role | Sees These Modules | Admin Modules Hidden |
|------|------------------|-----------------|
| **Admin** | All modules including Settings | None |
| **Project Lead** | Dashboard, Projects, Docs, Audit, Training, Risk, Accreditation, Supplier, Change Control | Data Hub, Workflow, Reports, User Mgmt |
| **Team Member** | Dashboard, Projects, Training, Accreditation, Messaging, Documents | Admin modules + specialized management |
| **Auditor** | Dashboard, Audit, Accreditation, Risk, Analytics, Report Builder | User/Data/Workflow management |

**If a module is missing:**
- Ask your **Administrator** to assign the role
- Non-admin users cannot access `/settings` admin sections
- Attempting restricted access redirects you to Dashboard

### Tool 5: Root Cause Analysis

**Purpose:** Perform structured root cause analysis using the **5 Whys** or **Fishbone diagram** approach

**When to Use:**
- Investigating incidents or near-misses
- Understanding why audit findings occurred
- Preventing repeat non-conformances
- Improving processes

**How to Use:**
1. Open **Chatbot**
2. Ask: "Perform root cause analysis for [incident]" (e.g., "Perform root cause analysis: Patient fall in ICU on March 1")
3. Provide:
   - Incident title
   - Detailed description
   - Category (clinical, administrative, operational, training, system)
   - Any initial findings
4. The AI performs analysis using:
   - **5 Whys technique** — drilling down to root causes
   - **Fishbone analysis** — identifying contributing factors across people, process, tools, environment
5. Output includes:
   - Root cause identification
   - Contributing factors
   - Corrective action recommendations

**Example Workflow:**
```
You: "RCA: Infection Control breach — glove storage not compliant"
AI: "ROOT CAUSE ANALYSIS - 5 Whys Method
     
     Issue: Gloves stored in non-sterile area
     
     Why 1: Storage cabinet out of service for 2 weeks
     Why 2: Maintenance order delayed due to budget approval
     Why 3: No expedited process for urgent supplies
     Why 4: Purchase policy lacks emergency clause
     Why 5: Process not designed for operational flexibility
     
     ROOT CAUSE: Rigid purchasing policy without emergency flexibility
     
     Contributing Factors (Fishbone):
     - People: Staff unfamiliar with interim procedures
     - Process: No emergency supply protocol
     - Tools: Old storage cabinet unreliable
     - Environment: Limited backup storage space
     
     Recommended Actions:
     1. Update purchase policy with emergency clause (Lead: Procurement)
     2. Identify interim storage options (Lead: Facilities)
     3. Train staff on backup procedures (Lead: Infection Control)
     4. Schedule preventive maintenance (Lead: Facilities)"
```

- Module hidden by organization/module settings

- Network delay
